# Next Phase Feature Ideation & Prioritization

> MVP 완성 이후, 전환율(면접 결정)과 체류시간을 높이기 위한 다음 단계 기능 기획
> PM/Designer/Engineer 3관점 아이디어 발산 → Impact/Effort 기반 수렴
> 기존 Discovery 결과물 (ICP, 페르소나, 여정맵, 가치제안) + 구현 현황 기반

---

## 현재 상태 요약 (Baseline)

### 완료된 기능
- 전 페이지 구현 (Home, Projects, Blog, About, 404)
- Featured Projects + 케이스 스터디
- 블로그 태그 필터 시스템
- 관련 글 추천 (태그 기반)
- 다크모드 토글 (system/light/dark)
- RSS 피드 (/feed.xml)
- 301 리다이렉트 (/posts → /blog)
- SEO (메타데이터, sitemap, robots.txt, canonical)
- 디자인 오버홀 (Terracotta OKLCH, Young Serif + Karla, 에디토리얼 레이아웃)
- 수치 하이라이트 (About)
- staggered fade-in 애니메이션

### 미해결 기회 (원래 Nice-to-Have에서 이월)
- OG 이미지 동적 생성 (E6)
- 추천사/소셜 프루프 섹션 (P6)
- 프로젝트 카테고리 필터 (P3)
- 미니멀 애니메이션 확장 (D7)

## Part 1: 브레인스토밍 (발산)

### PM 관점 — "어떤 기능이 면접 전환율을 더 올릴 수 있는가?"

| # | 아이디어 | 근거 |
|---|---------|------|
| P1 | **동적 OG 이미지 생성** (글/프로젝트별 자동 생성) | 채용담당자가 팀 슬랙에 링크 공유 시 시각적 품질이 클릭률 결정. 현재는 기본 이미지 하나만 사용 |
| P2 | **이력서/CV PDF 다운로드** (/about 하단) | 채용담당자 Journey Stage 5: 면접 결정 직후 이력서가 필요. 사이트에서 바로 다운로드 가능하면 전환 마찰 감소 |
| P3 | **프로젝트 성과 수치 시각화** (차트/그래프) | "800초→33초"를 텍스트로 쓰는 것 vs 막대그래프로 보여주는 것의 인지 차이. 수치의 임팩트 극대화 |
| P4 | **추천사/동료 피드백 섹션** | mattwilldev 벤치마크. "같이 일했던 사람의 평가"는 채용담당자에게 가장 강력한 소셜 프루프 |
| P5 | **콘텐츠 시리즈 기능** (연결된 글 묶음) | 테크리드(박준혁)의 연쇄 탐색 유도. "Vite 마이그레이션 Part 1/2/3" 같은 시리즈가 체류시간 증가 |

### Designer 관점 — "어떤 경험이 체류시간과 만족도를 높이는가?"

| # | 아이디어 | 근거 |
|---|---------|------|
| D1 | **블로그 글 목차 (TOC) 사이드바** | 긴 케이스 스터디/기술 글에서 구조를 한눈에 파악. 테크리드가 관심 섹션으로 바로 점프 가능 |
| D2 | **이미지 라이트박스** (클릭 확대) | 프로젝트 스크린샷, 아키텍처 다이어그램을 크게 보고 싶은 욕구. 현재 이미지가 고정 크기로만 표시 |
| D3 | **읽기 진행률 표시기** (프로그레스 바) | 긴 글을 읽을 때 "얼마나 남았는지" 시각적 피드백. 이탈 방지 + 완독률 향상 |
| D4 | **페이지 전환 애니메이션** | 라우트 간 부드러운 전환으로 "직접 만든 사이트"라는 인상 강화. 현재는 전환 시 하드컷 |
| D5 | **프린트 스타일시트** | 채용담당자가 프로젝트 케이스 스터디를 인쇄해서 면접위원회에 공유하는 시나리오. 실제로 발생하는 행동 패턴 |

### Engineer 관점 — "기술적으로 무엇이 품질과 확장성을 높이는가?"

| # | 아이디어 | 근거 |
|---|---------|------|
| E1 | **MDX 지원** (마크다운 + React 컴포넌트) | 인터랙티브 코드 데모, 임베디드 차트 등 기술 글의 표현력 확장. 테크리드에게 "이 사람은 콘텐츠도 잘 만드네" 인상 |
| E2 | **전체 텍스트 검색** (클라이언트 사이드) | 콘텐츠가 10개+ 넘어가면 태그 필터만으로는 부족. 키워드 기반 빠른 검색 |
| E3 | **Lighthouse CI / Web Vitals 모니터링** | 프론트엔드 개발자 포트폴리오인데 Core Web Vitals가 나쁘면 역설. 자동 성능 모니터링으로 품질 유지 |
| E4 | **E2E 테스트 (Playwright)** | 콘텐츠 추가/디자인 변경 시 주요 경로가 깨지지 않는지 자동 검증. 품질 보증 |
| E5 | **Vercel Analytics / 커스텀 이벤트 트래킹** | 어떤 프로젝트가 가장 많이 클릭되는지, 블로그 완독률은 얼마인지 데이터 기반 개선 |

## Part 2: 우선순위 매트릭스 (수렴)

### 평가 기준
- **Impact** (1~5): ICP 전환(면접 결정) + 체류시간에 미치는 영향
- **Effort** (1~5): 개발 시간/복잡도 (1=쉬움, 5=어려움)
- **Score**: Impact / Effort (높을수록 우선)

## Part 3: 구현 로드맵

### Wave 1 — Quick Wins (1~2일)
> 적은 노력으로 즉시 체감할 수 있는 개선. 바로 시작.

| 기능 | 설명 | 예상 시간 |
|------|------|----------|
| 읽기 진행률 표시기 (D3) | 블로그 상세 페이지 상단에 스크롤 기반 프로그레스 바 | 1시간 |
| 이미지 라이트박스 (D2) | 블로그/프로젝트 이미지 클릭 시 모달 확대 | 2시간 |
| 이력서 PDF 다운로드 (P2) | /about에 PDF 다운로드 링크 추가 | 30분 |
| 프린트 스타일시트 (D5) | @media print CSS 추가. 불필요 요소 숨김, 여백 조정 | 1시간 |

### Wave 2 — Core Enhancement (3~5일)
> 3개 ICP 모두에게 의미 있는 핵심 개선.

| 기능 | 설명 | 예상 시간 |
|------|------|----------|
| 블로그 글 목차 (D1) | 마크다운 H2/H3 파싱 → 사이드바 또는 글 상단 TOC. 스크롤 위치 하이라이트 | 4시간 |
| 동적 OG 이미지 (P1) | `next/og` (Satori) 활용. 글 제목 + 태그가 포함된 커스텀 OG 이미지 자동 생성 | 4시간 |
| Analytics 이벤트 (E5) | GA4 커스텀 이벤트: 프로젝트 클릭, 블로그 완독, CTA 클릭, 다크모드 전환 | 3시간 |
| 콘텐츠 시리즈 (P5) | frontmatter에 `series` 필드 추가 → 시리즈 내 이전/다음 링크 자동 생성 | 3시간 |

### Wave 3 — Advanced (1~2주)
> 콘텐츠가 충분해진 이후 진행. 기술적 투자 + 장기 가치.

| 기능 | 설명 | 예상 시간 |
|------|------|----------|
| MDX 지원 (E1) | next-mdx-remote 도입. 인터랙티브 코드 데모, 커스텀 컴포넌트 삽입 | 1일 |
| 전체 텍스트 검색 (E2) | fuse.js 클라이언트 검색. 빌드 타임에 인덱스 생성 | 4시간 |
| 추천사 섹션 (P4) | 동료/리드의 짧은 추천사 수집 → 홈 또는 About에 배치 | 콘텐츠 수집 의존 |
| E2E 테스트 (E4) | Playwright로 주요 경로 (홈→프로젝트→상세, 홈→블로그→글) 테스트 | 1일 |
| 페이지 전환 애니메이션 (D4) | View Transitions API 또는 framer-motion 라우트 전환 | 4시간 |

## Part 4: 핵심 가정 & 검증 방법

### Wave 1~2 실행 전 검증할 가정

| 가정 | 검증 방법 | 성공 기준 |
|------|----------|----------|
| OG 이미지가 링크 공유 시 클릭률을 높인다 | 카카오톡/슬랙에 링크 공유 후 시각적 확인 + GA4 유입 경로 비교 | 공유된 링크에서의 유입률 측정 가능 |
| TOC가 긴 글의 이탈률을 낮춘다 | GA4 스크롤 깊이 이벤트 비교 (TOC 추가 전/후) | 평균 스크롤 깊이 60%→75%+ |
| 태그 필터가 실제로 사용된다 | GA4 커스텀 이벤트로 태그 클릭 추적 | 블로그 방문자의 15%+ 태그 사용 |
| 이력서 PDF가 다운로드된다 | GA4 파일 다운로드 이벤트 | 월 5회+ 다운로드 |

## Part 5: 콘텐츠 로드맵 (기능과 병행)

> 기능만 있으면 빈 그릇. 콘텐츠가 기능의 가치를 결정.

### 단기 (1~2주)
| 콘텐츠 | 대상 ICP | 설명 |
|--------|---------|------|
| 블로그 3~5편 추가 | 테크리드, 동료 개발자 | FSD 도입기, 모노레포 전환, Playwright 테스트 전략 등 의사결정 중심 |
| 프로젝트 케이스 스터디 보강 | 채용담당자 | 기존 3개 프로젝트에 스크린샷, 아키텍처 다이어그램, 수치 데이터 추가 |

### 중기 (1~2개월)
| 콘텐츠 | 대상 ICP | 설명 |
|--------|---------|------|
| "AI 시대의 개발자" 시리즈 | 동료 개발자 | Cursor Skills 30+개 설계 경험, 에이전트 코딩 워크플로 |
| 추천사 수집 | 채용담당자 | 전 동료/리드에게 짧은 추천 메시지 요청 |

## 핵심 결론

**Wave 1 (Quick Wins)부터 즉시 시작:**
- 읽기 진행률, 이미지 라이트박스, 이력서 PDF, 프린트 CSS
- 총 5시간 이내, 체감 효과 즉각적

**Wave 2 (Core Enhancement)가 실질적 전환율 개선:**
- OG 이미지 = 공유 클릭률
- TOC = 긴 글 이탈률 감소
- Analytics = 데이터 기반 의사결정 시작

**콘텐츠가 기능보다 중요:**
- 현재 블로그 2편, 프로젝트 3개 → 최소 블로그 5편 이상이어야 태그 필터, 검색, 시리즈 기능이 의미를 가짐
- 기능 개발과 콘텐츠 작성을 병행해야 함
