# Feature Ideation & Prioritization

> PM/Designer/Engineer 3관점 아이디어 발산 → Impact/Effort 기반 수렴
> Discovery 결과물 (ICP, 페르소나, 여정맵, 가치제안)을 기반으로 필터링

## Part 1: 브레인스토밍 (발산)

### PM 관점 — "어떤 기능이 전환(면접 결정)에 가장 영향을 주는가?"

| # | 아이디어 | 근거 |
|---|---------|------|
| P1 | **프로젝트 케이스 스터디 페이지** (문제→해결→결과 구조) | 채용담당자 Journey Stage 4의 핵심. 면접 결정의 직접 근거 |
| P2 | **Featured Projects 섹션** (수치 성과가 보이는 카드) | Journey Stage 3에서 30초 안에 "이 사람 괜찮다" 판단 유도 |
| P3 | **프로젝트 카테고리 필터** (경력/사이드 구분) | 채용담당자는 "실무"를, 테크리드는 "성장 잠재력"을 보고 싶음 |
| P4 | **블로그 태그 시스템** | 테크리드가 관심 주제(아키텍처, 성능 등)로 빠르게 필터링 |
| P5 | **수치 하이라이트 섹션** (About 또는 랜딩) | 3초 신뢰를 위한 "800초→33초", "947 커밋" 같은 수치 시각화 |
| P6 | **추천사/소셜 프루프** 섹션 | mattwilldev 벤치마크. 동료/리드의 짧은 추천이 신뢰도를 높임 |

### Designer 관점 — "어떤 경험이 첫인상과 체류를 결정하는가?"

| # | 아이디어 | 근거 |
|---|---------|------|
| D1 | **Hero 섹션 리디자인** (태그라인 + 미니멀 비주얼) | 3초 룰. 시각적 품질 = 능력의 프록시 |
| D2 | **다크모드 토글** | 이미 CSS 토큰 준비됨. 개발자 사이트에서 기대되는 기능 |
| D3 | **반응형 완벽 대응** (모바일 우선) | 채용담당자 50%+ 모바일에서 확인. 프론트엔드인데 반응형 안 되면 치명적 |
| D4 | **프로젝트 카드 호버 인터랙션** | 탐색 단계에서 시각적 피드백 → 클릭 유도 |
| D5 | **글 읽기 경험 개선** (코드 하이라이팅, 이미지 확대) | 테크리드 체류시간 증가 → 글의 깊이를 더 느낌 |
| D6 | **경력 타임라인 비주얼** (About 페이지) | 3년의 궤적을 시각적으로 보여주면 성장 속도가 느껴짐 |
| D7 | **미니멀 애니메이션** (페이지 전환, 스크롤 등장) | 과하지 않되, "직접 만든 사이트"라는 인상 강화 |

### Engineer 관점 — "기술적으로 무엇이 필요하고, 무엇이 브랜드를 보여주는가?"

| # | 아이디어 | 근거 |
|---|---------|------|
| E1 | **라우트 구조 변경** (/posts→/blog, /projects 추가) | 아키텍처 기본. 이게 없으면 다른 기능을 붙일 곳이 없음 |
| E2 | **프로젝트 마크다운 시스템** (gray-matter 확장) | 기존 블로그와 동일 패턴으로 프로젝트 콘텐츠 관리 |
| E3 | **sitemap.ts 확장** (프로젝트, 블로그 모두 포함) | SEO 기본. 동료 개발자 유입 채널 |
| E4 | **301 리다이렉트** (/posts/[id] → /blog/[slug]) | 기존 SEO 자산 보존 |
| E5 | **RSS 피드 생성** | 동료 개발자 구독 채널. 장기적 네트워크 효과 |
| E6 | **OG 이미지 동적 생성** (글/프로젝트별) | 공유 시 시각적 품질. SNS에서의 클릭률 |
| E7 | **성능 최적화** (LCP < 1.5초 목표) | Journey에서 "2초 넘으면 이탈". 프론트엔드 개발자의 기본 |
| E8 | **관련 글 추천** (태그 기반) | 이수진 페르소나의 연쇄 탐색 유도 |

## 핵심 결론

**MVP는 "3가지 경로"가 작동하는 최소 단위:**

1. 채용담당자 경로: `/` → Featured Projects → `/projects/[slug]` (케이스 스터디)
2. 테크리드 경로: `/blog` → `/blog/[slug]` (기술 글)
3. 동료 개발자 경로: 검색 유입 → `/blog/[slug]` → 연쇄 탐색

이 3가지가 모두 작동하면 MVP 완성. 나머지는 점진적 개선.
